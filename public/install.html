Install

Up and running
in two minutes.

One install command, one start command. No API keys, no config files, no accounts to create. Briefings land automatically every weekday at 09:00 ET.

Deployment guide → Paper Trading
  1. 01

    Check requirements

    Peach requires Python 3.9 or later and git. Both come pre-installed on modern macOS. On Linux, install them with your package manager.

    # Verify versions
python3 --version   # need 3.9 or later
git --version
  2. 02

    Install Peach

    The install script clones the repository, creates a virtual environment, installs dependencies, and puts the peach command in ~/.local/bin. It takes about 30 seconds.

    curl -fsSL https://raw.githubusercontent.com/SkittlesGod-cmd/Peach-Agent/main/install.sh | bash

    The script installs to ~/.local/share/peach-agent/. Nothing goes in /usr/local or anywhere system-wide. Uninstalling is just deleting that directory.

  3. 03

    Start the agent

    This launches Peach as a detached background daemon. It schedules the morning briefing, starts the price-alert checker, and — if configured — starts the Discord bot.

    peach start

    Check that it's running:

    peach status
  4. 04

    Read your first briefing

    Until email or Discord is configured, briefings are written to a Markdown file after each 09:00 ET run. You can also trigger one immediately:

    # Run a briefing right now instead of waiting for 09:00
peach stop
PEACH_RUN_ON_START=true peach start

# Read the output
cat ~/.local/share/peach-agent/briefing.md
  5. 05

    Try Paper Trading New

    Log hypothetical long and short positions before committing real capital. Peach tracks live P&L and records your performance when you close.

    Paper trading guide →
  6. 06

    Add email delivery Optional

    Edit ~/.local/share/peach-agent/peach_config.json and add your SMTP credentials. Gmail with an App Password works well.

    {
  "smtp_username": "you@gmail.com",
  "smtp_password": "your-app-password",
  "email_from":    "you@gmail.com",
  "email_to":      "you@gmail.com"
}

    Restart to apply:

    peach stop && peach start

    Gmail requires an App Password, not your account password. Generate one at Google Account → Security → 2-Step Verification → App passwords.

  7. 07

    Customise your watchlist Optional

    The default watchlist is SPY, QQQ, DIA, IWM, AAPL, MSFT, NVDA, TSLA. Replace it with tickers you actually follow:

    {
  "tickers": ["SPY", "QQQ", "AAPL", "MSFT", "GOOGL", "META", "AMZN"]
}

CLI reference

peach start Start the background daemon. Returns immediately — the daemon keeps running.
peach status Show whether the daemon is running, the PID, and where logs are written.
peach stop Stop the daemon gracefully. Cleans up the PID file.
PEACH_RUN_ON_START=true
peach start		 Run a briefing immediately on startup instead of waiting for the scheduled time.
peach --home /path start Use a custom directory for config, PID, and log files. Useful for multiple instances.

What the briefing contains

Every morning briefing includes five sections, generated by the LLM from the market data payload:

  • Macro Tape — index levels and overnight moves across the configured watchlist
  • Portfolio Check — open positions benchmarked against current prices (if configured)
  • Standouts — unusual price and volume moves from the watchlist
  • Lookout List — 2–3 tickers worth watching that session, with data-backed reasons
  • Risk Notes — explicit uncertainty, data gaps, and what could go wrong

If the LLM is unreachable, Peach produces a deterministic data-first fallback and marks it clearly so you know the difference.
