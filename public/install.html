Install

Up and running
in five minutes.

One install command, one start command, one hunt command. No API keys, no accounts. Peach reads your student profile and starts finding professors.

View dashboard → Deployment guide
  1. 01

    Check requirements

    Peach requires Python 3.9 or later and git. Both come pre-installed on modern macOS. On Linux, install them with your package manager.

    # Verify versions
python3 --version   # need 3.9 or later
git --version
  2. 02

    Install Peach

    The install script clones the repository, creates a virtual environment, installs dependencies, and puts the peach command in ~/.local/bin. It takes about 30 seconds.

    curl -fsSL https://raw.githubusercontent.com/SkittlesGod-cmd/Peach-Agent/main/install.sh | bash

    Installs to ~/.local/share/peach-agent/. Nothing system-wide. Uninstalling is deleting that directory.

  3. 03

    Set your student profile

    Edit ~/.local/share/peach-agent/peach_config.json and fill in your profile. Peach uses this to write personalized emails that reference your exact background.

    {
  "email_to": "you@gmail.com",
  "student_profile": {
    "name":      "Your Name",
    "grade":     "11th grade",
    "gpa":       "4.0",
    "courses":   "AP Biology, AP Chemistry, AP Computer Science",
    "skills":    "Python, data analysis, literature review",
    "interests": "neurosurgery, brain tumors, computational neuroscience",
    "email":     "you@gmail.com"
  }
}

    The email_to field is the address where Peach confirms each sent email. Set student_profile.email to the address professors will reply to — usually the same.

  4. 04

    Start the daemon

    This launches Peach as a detached background process. It handles scheduled hunts and follow-up automation.

    peach start

# Confirm it's running
peach status
  5. 05

    Run your first research hunt

    This searches 3 universities for neuroscience and neurosurgery professors, scrapes their lab pages, and sends personalized emails to any whose address appears on their page.

    # Search 3 universities (expand --batch to go wider)
peach research hunt --batch 3

# See who was found
peach research list

# Sync results to your web dashboard
peach research sync

    The hunt runs in real time — you'll see tool calls logged as the agent searches, scrapes, and emails. Run with --batch 5 to cover more universities per run.

  6. 06

    Check for follow-ups Weekly

    After 14 days, professors who haven't replied are surfaced automatically. Run this weekly to keep your outreach pipeline active.

    peach research followup
  7. 07

    View the dashboard Optional

    The research dashboard shows every professor found, emailed, and replied. Requires Vercel KV — set it up in the Vercel dashboard under Storage → Create KV database → connect to your project.

    Open dashboard →

Research CLI reference

peach research hunt Search 3 universities for professors, scrape lab pages, send outreach emails.
peach research hunt --batch 5 Cover 5 universities per run. Maximum recommended is 8 (takes ~10 min).
peach research followup Send polite follow-ups to professors who haven't replied in 14+ days.
peach research list Print all professors in the database with their status.
peach research list --status emailed Filter by status: found, emailed, followed_up, replied, or declined.
peach research sync Push the full professor database to the Vercel dashboard.

Daemon CLI reference

peach start Start the background daemon. Returns immediately — the daemon keeps running.
peach status Show whether the daemon is running, the PID, and where logs are written.
peach stop Stop the daemon gracefully. Cleans up the PID file.
peach --home /path start Use a custom config/PID/log directory. Useful for multiple instances.

What happens during a hunt

When you run peach research hunt, the agent executes a full agentic loop:

  • Web search — queries DuckDuckGo for real faculty directory pages at each target university
  • Lab page scraping — fetches each promising URL and extracts email addresses, paper titles, and research focus
  • Database entry — saves each professor to a local SQLite database (duplicates are skipped)
  • Outreach email — if a real email was found on the page, drafts and sends a personalized cold email under 200 words
  • Dashboard sync — pushes the updated database to your Vercel dashboard after the hunt completes

The entire pipeline runs in one command. Each hunt covers 3 universities by default and takes 3–8 minutes depending on network speed.
