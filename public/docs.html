Install and run
the agent.

The project is a Python package with a console command. Install it, configure credentials, and let the daemon handle the weekday briefing run.

Quick Install

Run this on any machine with Python 3.9+ and git. The script clones the repository, creates a virtual environment, installs dependencies, and drops a peach command in ~/.local/bin.

curl -fsSL https://raw.githubusercontent.com/SkittlesGod-cmd/Peach-Agent/main/install.sh | bash

Configure Credentials

Peach reads environment variables first, then falls back to peach_config.json. Set these before starting:

# Email delivery (required)
export SMTP_USERNAME="you@example.com"
export SMTP_PASSWORD="your-app-password"
export PEACH_EMAIL_FROM="you@example.com"
export PEACH_EMAIL_TO="you@example.com"

# LLM provider — OpenAI:
export PEACH_LLM_PROVIDER="openai"
export OPENAI_API_KEY="sk-..."

# — or Ollama (local, no API key needed):
export PEACH_LLM_PROVIDER="ollama"
export OLLAMA_URL="http://127.0.0.1:11434"

Credentials can also live in peach_config.json inside the install directory (~/.local/share/peach-agent/). Environment variables always take precedence.

Run

peach start     # start the background daemon
peach status    # check if it's running and where logs are
peach stop      # stop the process and clean up the PID file

To trigger a briefing immediately without waiting for 09:00 ET:

PEACH_RUN_ON_START=true peach start

Manual Install (from source)

git clone https://github.com/SkittlesGod-cmd/Peach-Agent.git
cd Peach-Agent
python3 -m venv .venv-peach
source .venv-peach/bin/activate
pip install -r requirements-peach.txt
pip install .
cp peach_config.example.json peach_config.json

What Peach Sends

  • Macro tape overview derived from the supplied market payload.
  • Unusual price and volume standouts from the configured ticker list.
  • Headline-backed market sentiment notes.
  • A 2–3 ticker lookout list with technical justifications from observed metrics.
  • Risk notes and explicit uncertainty when APIs or LLM providers are unavailable.

If the LLM provider is unreachable, Peach generates a deterministic briefing from raw metrics. A provider outage will never crash the daemon.
