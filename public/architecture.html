System map

Small modules, clear responsibilities.

Peach is split into files that match the operational pipeline, which makes it easier to test, replace providers, and deploy outside a laptop.

Core Modules

ModuleResponsibility
main.pyArgparse command controller for start, status, and stop.
daemon.pyPersistent scheduler and guarded execution loop.
data_fetcher.pyMarket metrics and headline aggregation.
analyzer.pyOpenAI/Ollama prompt construction and fallback analysis.
notifier.pyMarkdown email formatting and SMTP delivery.

Failure Handling

Each network-sensitive stage is wrapped with explicit exception handling and logging. A failed fetcher or provider can degrade the report, but it should not terminate the daemon.

Runtime Files

  • .peach.pid tracks the detached process.
  • peach.log stores rotating operational logs.
  • peach_config.json stores local non-committed configuration.
