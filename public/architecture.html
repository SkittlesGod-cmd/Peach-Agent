Peach is split into files that map directly to the operational pipeline. Every module does one thing. The agent loop composes them.

The agentic loop

The core idea: the LLM doesn't just fill in a template. It reasons, decides what data it needs, calls tools, reads the results, and continues until it has a complete answer.

Perception   →   Reason   →   Tool call   →   Re-reason   →   Output
     ↑                                                              ↓
     └──────────────────── Feedback loop ──────────────────────────┘

Tools available (9 total):
  get_quote              current price, change, volume
  get_portfolio          positions with live P&L
  get_pre_market         pre-market price and % change
  get_news               recent headlines for a ticker
  get_technicals         RSI(14), SMA20/50, 52-week H/L
  get_earnings_calendar  watchlist tickers reporting in next 7 days
  get_correlation        30-day return correlation matrix
  search_web             real-time DuckDuckGo search
  add_alert              set a price threshold alert

Each conversation or briefing run allows up to 8 tool iterations. If the agent needs pre-market data for three tickers and portfolio context, it calls four tools in sequence before writing the briefing. The LLM is retried up to 3 times with exponential backoff before falling back to MarketAnalyzer.

Core modules

ModuleResponsibility
main.py CLI controller for start, status, stop, run, logs, upgrade, doctor. Manages the PID file and spawns the daemon as a detached subprocess.
daemon.py Persistent APScheduler loop with four jobs: morning briefing (weekdays 09:00), alert checker (every 5 min, 09:00–16:00), portfolio snapshot (weekdays 16:05), and correlation report (Mondays 09:35). Handles SIGTERM/SIGINT for clean shutdown.
agent.py Agentic reasoning loop. Sends messages to the LLM, executes tool calls, feeds results back, and repeats up to 8 times per run. Falls back to MarketAnalyzer on error.
tools.py Tool schema definitions (OpenAI function-calling format) and ToolExecutor — the dispatcher that runs each tool and returns a string result.
data_fetcher.py Batch market data collection. Fetches previous-session closes, pre-market prices, volumes, and headlines for the configured watchlist. Used by both the daemon and the agent.
analyzer.py One-shot LLM analysis fallback. Used when the agentic loop is unavailable or for providers that don't support tool use. Supports peach proxy, OpenRouter, OpenAI, and Ollama.
portfolio.py SQLite-backed position and alert ledger. Positions are averaged on re-add. Alerts are checked every 5 minutes; each fires once and is marked done.
memory.py SQLite key-value store for persistent agent context — Discord channel ID, user preferences, and any facts the agent should remember across sessions.
discord_bot.py Discord bot interface. Runs in a daemon thread with its own asyncio loop. Handles commands with a ! prefix (!briefing, !chart, !peach, !trade, !trades, !closetr, etc.) and pushes morning briefs, charts, and alerts to the configured channel.
charts.py Chart generation via mplfinance and matplotlib. Produces OHLCV candlestick charts, portfolio P&L bars, sector allocation pie, correlation heatmap, portfolio history line, and multi-ticker comparison overlay. Returns PNG bytes for Discord delivery or PDF embedding.
report.py PDF morning brief generator using fpdf2. Sections: macro snapshot (VIX/10Y/DXY), portfolio positions table, embedded chart images, and full LLM analysis text. Returns PDF bytes.
cache.py Thread-safe TTL cache for yfinance responses. Prevents repeated API hits during the tool-use loop. TTL: 60 s for ticker info, 300 s for historical data.
notifier.py Formats the briefing as a multipart email (plain + HTML) and delivers via SMTP with STARTTLS. Always writes briefing.md regardless of delivery success.
config.py Loads configuration from env vars and peach_config.json. Returns a frozen PeachConfig dataclass. Env vars always win.

Failure handling model

Every network-sensitive stage is individually guarded. Failures degrade the output gracefully — they never terminate the daemon:

  1. 1

    Individual headline fetchers

    If Alpha Vantage fails, Peach tries NewsAPI. If that fails, it falls back to Yahoo Finance RSS. The briefing always has headlines.

  2. 2

    Individual ticker fetches

    If yfinance returns no data for one ticker, a null-filled StockMetric is appended. The remaining tickers still run. Null fields are reported explicitly in the briefing.

  3. 3

    Agentic loop failure

    If the tool loop raises an exception, PeachAgent.run_briefing catches it and calls MarketAnalyzer as a one-shot fallback. The briefing is still delivered.

  4. 4

    LLM provider unreachable

    If the LLM API call fails, MarketAnalyzer generates a deterministic data-first briefing from raw metrics. The email is sent; the report is marked as a fallback.

  5. 5

    Email delivery failure

    If SMTP raises, the exception is logged with a full traceback. The daemon stays alive. The next scheduled run will attempt delivery again.

LLM proxy architecture

The default provider is a Vercel serverless function (api/analyze.js) that forwards requests to OpenRouter. The real API key lives only in Vercel's encrypted environment variables — it is never in git, never on the user's machine.

User's machine                  Vercel                  OpenRouter
┌─────────────────┐            ┌────────────┐           ┌──────────┐
│ PeachAgent      │  POST      │ /api/      │  POST +   │ LLM API  │
│ _call_llm()     │ ─────────► │ analyze.js │  Bearer   │          │
│ (no API key)    │            │            │ ─────────►│          │
└─────────────────┘            └────────────┘           └──────────┘
                                  Key lives here only

This means a user can install Peach and get a working LLM-backed briefing with zero configuration — no API accounts, no keys to manage.
