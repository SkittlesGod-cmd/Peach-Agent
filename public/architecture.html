Architecture

Small modules,
clear responsibilities.

Peach is split into files that map directly to the operational pipeline — easier to test, replace a provider, or move to a new host.

Core Modules

ModuleResponsibility
main.pyArgparse CLI controller for start, status, and stop. Manages the PID file and spawns the daemon as a detached subprocess.
daemon.pyPersistent APScheduler loop. Handles SIGTERM/SIGINT for clean shutdown. Runs the pipeline as a guarded job with a 1-hour misfire grace window.
data_fetcher.pyMarket metrics via yfinance (close, previous close, volume, % change). Headlines via Alpha Vantage, NewsAPI, or Yahoo Finance RSS in fallback order.
analyzer.pyBuilds the analyst prompt and dispatches to OpenAI or Ollama. Falls back to a deterministic template report if the LLM provider is unreachable.
notifier.pyFormats the Markdown report as a multipart email (plain + HTML) and delivers via SMTP with STARTTLS. Falls back to a basic HTML renderer if the markdown package is missing.
config.pyLoads configuration from environment variables and peach_config.json. Env vars take precedence. Provides a frozen PeachConfig dataclass.
logging_setup.pyConfigures a rotating file handler (5 MB × 5 backups). Optionally mirrors to stdout when the CLI is run with --verbose.

Failure Handling Model

Every network-sensitive stage is wrapped in explicit exception handling. The hierarchy is:

  • A failed headline fetcher logs a warning and moves on — the remaining fetchers are still attempted.
  • A failed market metric for one ticker appends a null-filled record — the rest of the tickers still run.
  • A failed LLM call triggers the deterministic fallback briefing — the email is still sent.
  • A failed email send logs the exception — the daemon stays alive for the next scheduled run.

No single failure terminates the daemon process. All failures are logged to peach.log with full tracebacks.

Runtime Files

FilePurpose
.peach.pidTracks the detached daemon PID. Written by the CLI on spawn; cleaned up on graceful shutdown.
peach.logRotating operational log. All pipeline events, warnings, and exceptions are written here.
peach_config.jsonLocal non-committed configuration. Overlaid by environment variables at startup.

Provider Contracts

Swapping an LLM provider requires only changing PEACH_LLM_PROVIDER. The analyzer dispatches to the appropriate client inside MarketAnalyzer.analyze() — nothing else in the pipeline changes. The same pattern applies to news sources: the fetcher tries each source in order and stops when it has enough headlines.
