Configuration

Providers, tickers,
schedule, and mail.

Peach reads environment variables first, then falls back to peach_config.json in the Peach home directory. Every key below has a corresponding env var.

Email (Required)

export SMTP_USERNAME="you@example.com"
export SMTP_PASSWORD="your-app-password"
export PEACH_EMAIL_FROM="you@example.com"
export PEACH_EMAIL_TO="you@example.com"

# Optional overrides:
export SMTP_HOST="smtp.gmail.com"   # default
export SMTP_PORT="587"              # default
export PEACH_EMAIL_SUBJECT_PREFIX="Peach Market Briefing"

Market News Sources

News providers are tried in order. Peach moves to the next source when the previous one returns no usable headlines or is unconfigured.

SourceEnv VariableNotes
Alpha VantageALPHA_VANTAGE_API_KEYTicker-matched news sentiment feed. Best quality.
NewsAPINEWS_API_KEYGeneral financial news. Used if Alpha Vantage is unconfigured or fails.
Yahoo Finance RSSNone requiredAutomatic fallback. No API key needed.

Schedule and Tickers

{
  "timezone": "America/New_York",
  "schedule_hour": 9,
  "schedule_minute": 0,
  "tickers": ["SPY", "QQQ", "DIA", "IWM", "AAPL", "MSFT", "NVDA", "TSLA"],
  "headline_limit": 10,
  "run_on_start": false
}

All peach_config.json keys have environment variable equivalents. For example, schedule_hour maps to PEACH_SCHEDULE_HOUR. Env vars always win.

Peach Home

By default, Peach uses the current directory as its home. Pin it with PEACH_HOME or the --home flag:

peach --home /opt/peach start
peach --home /opt/peach status
peach --home /opt/peach stop
