Peach reads environment variables first, then falls back to peach_config.json in the Peach home directory. Every key below has a matching env var.

Deployment guide → Architecture

Minimal working config

With zero config, Peach works immediately — the LLM is pre-configured via the proxy and briefings land in ~/briefing.md. Add only what you need:

# ~/.local/share/peach-agent/peach_config.json
{
  "tickers":          ["SPY", "QQQ", "AAPL", "MSFT", "NVDA", "TSLA"],
  "discord_token":    "YOUR_BOT_TOKEN",
  "discord_channel_id": "YOUR_CHANNEL_ID"
}

Environment variables always win. PEACH_SCHEDULE_HOUR=8 overrides "schedule_hour": 9 in the JSON. This makes it easy to use config files for defaults and env vars for host-specific overrides.

Discord

Enable the Discord bot to get briefings, charts, and alerts pushed to a channel — and to use !peach, !chart, !technicals, !trade, and all other commands from Discord.

KeyEnv varDescription
discord_token DISCORD_TOKEN Bot token from the Discord Developer Portal. Required to enable the Discord bot.
discord_channel_id DISCORD_CHANNEL_ID Channel ID for proactive pushes — morning briefings, PDF, charts, and drawdown alerts. The bot must have permission to send messages and files in this channel. 
{
  "discord_token":      "YOUR_BOT_TOKEN",
  "discord_channel_id": "1234567890123456789"
}

LLM providers

The default provider is peach — a hosted proxy with a pre-configured OpenRouter key. No API key required. Switch providers if you want more control or a different model.

ProviderKey / Env varDefault model
peach (default) None — pre-configured anthropic/claude-3-5-haiku
openrouter OPENROUTER_API_KEY
PEACH_OPENROUTER_MODEL		 anthropic/claude-3-5-haiku
openai OPENAI_API_KEY
PEACH_OPENAI_MODEL		 gpt-4o-mini
ollama OLLAMA_URL
PEACH_OLLAMA_MODEL		 llama3.1 at 127.0.0.1:11434 
# Switch to OpenRouter with your own key and a smarter model:
{
  "llm_provider":      "openrouter",
  "openrouter_model":  "anthropic/claude-opus-4"
}
# Set your key as an env var (never in the JSON file):
export OPENROUTER_API_KEY=sk-or-...

Schedule & tickers

KeyEnv varDefaultDescription
tickers PEACH_TICKERS (CSV) SPY, QQQ, DIA, IWM… Watchlist for the briefing. Any valid Yahoo Finance ticker.
timezone PEACH_TIMEZONE America/New_York IANA timezone for the scheduler. The schedule hour is interpreted in this timezone.
schedule_hour PEACH_SCHEDULE_HOUR 9 Hour to run the briefing (24h, in the configured timezone).
schedule_minute PEACH_SCHEDULE_MINUTE 0 Minute offset within the hour.
headline_limit PEACH_HEADLINE_LIMIT 10 Maximum headlines to include in the market payload.
run_on_start PEACH_RUN_ON_START false Run a briefing immediately on daemon start, before the first scheduled run.
drawdown_alert_pct PEACH_DRAWDOWN_PCT 0.05 Drawdown threshold for position alerts. A notification fires when any position falls this percentage below its cost basis. Rate-limited to once per 24 hours per ticker.

Email delivery Optional

If all six email fields are present, Peach delivers via SMTP. Otherwise, the briefing is written to briefing.md in the Peach home directory.

{
  "smtp_host":             "smtp.gmail.com",
  "smtp_port":             587,
  "smtp_username":         "you@gmail.com",
  "smtp_password":         "your-app-password",
  "email_from":            "you@gmail.com",
  "email_to":              "you@gmail.com",
  "email_subject_prefix":  "Peach Market Briefing"
}

News sources Optional

News providers are tried in order until enough headlines are collected. Yahoo Finance RSS is always the final fallback and requires no key.

SourceEnv varQuality
Alpha Vantage ALPHA_VANTAGE_API_KEY Best — ticker-matched sentiment feed with summaries.
NewsAPI NEWS_API_KEY Good — general financial news, sorted by recency.
Yahoo Finance RSS Not required Automatic fallback. Always available, no key needed.

Peach home directory

All runtime files live in the Peach home directory. The default is ~/.local/share/peach-agent/ (set by the install script). Override with:

# Via env var (persistent):
export PEACH_HOME=/opt/peach

# Via flag (per-command):
peach --home /opt/peach start
peach --home /opt/peach status
FilePurpose
peach_config.jsonLocal configuration. Never committed to git.
peach.logRotating operational log (5 MB × 5 backups).
.peach.pidPID file tracking the running daemon.
briefing.mdLatest briefing, written when email is not configured.
portfolio.dbSQLite — positions, alerts, paper trades, and daily portfolio snapshots.
memory.dbSQLite key-value store for agent memory, chat ID, and drawdown rate limits.
morning_brief.pdfMost recent PDF morning brief — overwritten each run and delivered to Discord.
