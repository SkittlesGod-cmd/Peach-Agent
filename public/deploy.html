Deployment

Host the site on Vercel.
Run the daemon somewhere always-on.

This static site deploys to Vercel in one click. The briefing scheduler is a persistent Python process and needs a host that stays awake.

One-Line Install

Works on Linux and macOS. Requires Python 3.9+ and git.

curl -fsSL https://raw.githubusercontent.com/SkittlesGod-cmd/Peach-Agent/main/install.sh | bash

Good Hosts for the Daemon

HostNotes
Cloud VM (DigitalOcean, Linode, EC2)Simplest reliable path. A $4–6/month Ubuntu VPS is more than enough.
Home serverAny machine that stays powered and networked. Use systemd or launchd to restart on reboot.
Docker containerRun with --restart unless-stopped and pass credentials via --env-file.
Raspberry PiWorks well for a dedicated, always-on local host with minimal power draw.

Production Setup

# 1. Install Peach
curl -fsSL https://raw.githubusercontent.com/SkittlesGod-cmd/Peach-Agent/main/install.sh | bash

# 2. Set credentials (or edit ~/.local/share/peach-agent/peach_config.json)
export SMTP_USERNAME="you@example.com"
export SMTP_PASSWORD="your-app-password"
export PEACH_EMAIL_FROM="you@example.com"
export PEACH_EMAIL_TO="you@example.com"
export PEACH_LLM_PROVIDER="openai"
export OPENAI_API_KEY="sk-..."

# 3. Start
peach start
peach status

Vercel runs serverless functions triggered by HTTP requests. It is not a substitute for a continuously running background scheduler. The daemon needs a persistent process on an always-on host.

Keeping It Running After Reboots

On Linux with systemd, create a unit file that starts Peach on boot and restarts it on failure. The peach command installed by the script works directly as the ExecStart target.

On macOS, use a launchd plist in ~/Library/LaunchAgents/ to achieve the same effect.
