Deployment

Run it anywhere
you want.

Peach runs on any machine that stays on. A $4/month VPS, a home server, a Docker container, or a Raspberry Pi — the install command is the same everywhere.

Vercel (which hosts this site) runs serverless functions on request — it cannot run a persistent background scheduler. The briefing daemon needs a process that stays alive between 09:00 ET each weekday. Use any of the options below.

The same two commands, everywhere

Peach installs identically on any Linux or macOS host. SSH in and run:

curl -fsSL https://raw.githubusercontent.com/SkittlesGod-cmd/Peach-Agent/main/install.sh | bash
peach start

Choose a host

  • Cloud VM

    Simplest

    DigitalOcean, Linode (Akamai), Vultr, or AWS EC2. A $4–6/month Ubuntu instance is more than sufficient. You get SSH access, full control, and a static IP for the Telegram webhook if you need it.

    # On a fresh Ubuntu 22.04 VPS:
sudo apt update && sudo apt install -y python3 python3-pip git
curl -fsSL https://raw.githubusercontent.com/SkittlesGod-cmd/Peach-Agent/main/install.sh | bash
peach start

  • Docker

    Portable

    Run Peach inside a container with --restart unless-stopped so it comes back after a host reboot. Pass config via --env-file.

    # Build from the repo
git clone https://github.com/SkittlesGod-cmd/Peach-Agent.git
cd Peach-Agent

# Create a .env file with your credentials
cat > peach.env <<EOF
TELEGRAM_BOT_TOKEN=your_token
SMTP_USERNAME=you@gmail.com
SMTP_PASSWORD=your-app-password
PEACH_EMAIL_FROM=you@gmail.com
PEACH_EMAIL_TO=you@gmail.com
EOF

# Run with Docker
docker run -d \
  --name peach \
  --restart unless-stopped \
  --env-file peach.env \
  -v peach-data:/root/.local/share/peach-agent \
  python:3.11-slim \
  bash -c "pip install -e /app[all] && peach start --foreground"

  • Home server

    No recurring cost

    Any always-on Mac or Linux machine at home works. Install Peach, then use systemd (Linux) or launchd (macOS) to start it on boot and restart it on failure.

  • Raspberry Pi

    Low power

    A Pi 3 or later running Raspberry Pi OS works great. Minimal power draw, completely silent, and it can double as a local Ollama server if you want local LLM analysis.

    # Install Python 3.9+ if needed
sudo apt install python3.11 python3.11-venv git
curl -fsSL https://raw.githubusercontent.com/SkittlesGod-cmd/Peach-Agent/main/install.sh | bash
peach start

Survive reboots with systemd (Linux)

Create a systemd service so Peach restarts automatically if the machine reboots or crashes:

# Create the unit file
sudo tee /etc/systemd/system/peach.service <<EOF
[Unit]
Description=Peach pre-market intelligence agent
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=$USER
ExecStart=$HOME/.local/bin/peach start --foreground
Restart=on-failure
RestartSec=30
Environment=PEACH_HOME=$HOME/.local/share/peach-agent

[Install]
WantedBy=multi-user.target
EOF

# Enable and start
sudo systemctl daemon-reload
sudo systemctl enable peach
sudo systemctl start peach
sudo systemctl status peach

Survive reboots with launchd (macOS)

On macOS, use a launchd plist in ~/Library/LaunchAgents/:

# Create the plist
cat > ~/Library/LaunchAgents/com.peach.agent.plist <<EOF
<?xml version="1.0" encoding="UTF-8"?>
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>com.peach.agent</string>
  <key>ProgramArguments</key>
  <array>
    <string>$HOME/.local/bin/peach</string>
    <string>start</string>
    <string>--foreground</string>
  </array>
  <key>RunAtLoad</key>
  <true/>
  <key>KeepAlive</key>
  <true/>
</dict>
</plist>
EOF

# Load it
launchctl load ~/Library/LaunchAgents/com.peach.agent.plist

Running in a different timezone? Set PEACH_TIMEZONE to your host's timezone (e.g. Europe/London) and keep schedule_hour: 9 — Peach will convert to local time correctly.
